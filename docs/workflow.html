HeavyLeads System Workflow

Lead generation platform for heavy equipment dealers. Scrapes public construction data, matches it to your inventory and location, and delivers scored leads daily.

1. The Big Picture

Scraper Pipeline runs daily at 06:00 UTC

8 adapters pull permits, bids, and news from public sources. Records are validated, geocoded, and upserted into the database. Cross-source deduplication merges records about the same project.

Leads are enriched on read

When you open the dashboard, each lead is analyzed: equipment types are inferred from project descriptions, a relevance score (0-100) is computed based on your profile, and construction phases are detected for timeline mapping.

You filter, bookmark, and track

Filter leads by equipment type, radius, keyword, date, and project value. Bookmark promising leads, track their status (new → contacted → won/lost), and save filter combos as Saved Searches.

Email digests deliver new matches daily

Saved Searches with digest enabled generate daily emails listing new leads from the last 24 hours that match your saved criteria.

2. Data Ingestion — The Scraper

How to trigger it

Leads don't exist until the scraper runs. Two ways:

curl -X POST http://localhost:3000/api/scraper/run

Or wait for the daily cron at 06:00 UTC (runs automatically when the dev server is up).

Pipeline Steps

1. Scrape

Each adapter fetches raw data from its source API or feed. Returns RawLeadData[].

2. Validate

Zod schema validation. Requires at least one of: permitNumber, title, or externalId. Invalid records logged and skipped.

3. Geocode

Google Maps API converts addresses to lat/lng. Throttled at 25ms between requests. Skipped if adapter already provides coordinates.

4. Upsert

Permits: upsert on (sourceId, permitNumber). Non-permits: check existing by (sourceId, title), insert if new. Creates leadSources entry for attribution.

5. Deduplicate

Cross-source dedup: finds leads within 0.1 miles of each other with >70% text similarity in address or title. Merges source entries and deletes the duplicate.

3. Lead Enrichment — What Happens on Read

Raw scraped data becomes actionable intelligence when you view the dashboard. Three systems run on every query:

Equipment Inference

10 keyword rules analyze projectType and description to determine what equipment a project needs.

Example: "excavation, earthwork, grading" → Excavators, Bulldozers, Compactors

Confidence: High (projectType match), Medium (description match), Low (fallback)

Timeline Mapping

Detects construction phases from the same keywords and maps them to urgency levels:

  • Now Site Prep, Foundation, Paving
  • Soon Framing, Structural, Exterior
  • Later Interior Finishing, Landscaping

Lead Scoring (0–100)

Each lead gets a relevance score based on your company profile:

Equipment Match
50 pts

Ratio of inferred equipment types that overlap with your inventory

Proximity
30 pts

Linear decay within service radius (closer = higher)

Project Value
20 pts

Logarithmic scale, $1M+ projects cap at 20 points

5. Lead Detail Page

URL: /dashboard/leads/[id]

Click any lead card to see the full detail view. Three-column grid:

Left Column

  • Location map placeholder
  • Full project details (address, city, state, dates, value, applicant, contractor, agency)

Right Column

6. Lead Tracking

Statuses

Track where each lead is in your pipeline:

  • New — Default, haven't looked at it
  • Viewed — Opened the detail page
  • Contacted — Reached out to the applicant
  • Won — Closed the deal
  • Lost — Didn't work out

Per-user, per-org. Team members see their own statuses.

Bookmarks

Toggle bookmark on any lead (card or detail page). View all bookmarked leads at:

/dashboard/bookmarks

Per-user, per-org. Quick way to build a shortlist of hot leads.

7. Saved Searches & Email Digests

Saving a Search

1. Apply filters on the dashboard

Set equipment types, radius, keyword, dates, project size to your liking.

2. Click "Save Search"

Redirects to /dashboard/saved-searches?save=true&[params] with a name input.

3. All filter params stored as database columns

Not a JSON blob — explicit columns allow SQL queries for digest generation. isDigestEnabled flag controls daily emails.

Email Digests

When isDigestEnabled = true on a saved search, the system generates a daily email with leads from the last 24 hours matching those filters. Uses Resend for delivery.

Trigger manually for testing:

curl -X POST http://localhost:3000/api/email-digest

8. User & Org Setup Flow

Sign Up /sign-up

Name, email, password, company name. Creates user + organization + Stripe customer. Sets org as active on the session.

Onboarding /onboarding

Step 1: HQ address (geocoded to lat/lng for distance calculations)
Step 2: Equipment types you sell/rent (used for scoring and matching)
Step 3: Service radius in miles (default 50, range 10–500)

Billing /billing

Stripe checkout for monthly subscription + one-time setup fee. In dev mode: "Skip (Dev Only)" button creates a mock subscription.

Dashboard /dashboard

Your daily lead feed. Requires: authenticated session + active organization + completed onboarding + active subscription.
