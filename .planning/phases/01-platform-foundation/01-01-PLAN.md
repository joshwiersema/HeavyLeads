  <action>
    1. Create `src/lib/auth.ts` -- Better Auth server configuration:
       ```typescript
       import { betterAuth } from "better-auth";
       import { organization } from "better-auth/plugins";
       import { nextCookies } from "better-auth/next-js";
       import { drizzleAdapter } from "better-auth/adapters/drizzle";
       import { db } from "./db";

       export const auth = betterAuth({
         database: drizzleAdapter(db, { provider: "pg" }),
         emailAndPassword: { enabled: true },
         plugins: [
           organization({
             creatorRole: "owner",
             membershipLimit: 50,
           }),
           nextCookies(), // MUST be last plugin
         ],
       });
       ```

    2. Create `src/lib/auth-client.ts` -- Better Auth client:
       ```typescript
       import { createAuthClient } from "better-auth/react";
       import { organizationClient } from "better-auth/client/plugins";

       export const authClient = createAuthClient({
         plugins: [organizationClient()],
       });

       // Export convenience hooks
       export const { useSession, signIn, signUp, signOut, useActiveOrganization } = authClient;
       ```

    3. Create `src/lib/validators/auth.ts`:
       - `signUpSchema`: email (email format), password (min 8 chars), name (min 2 chars), companyName (min 2 chars, for creating the org)
       - `signInSchema`: email (email format), password (min 1 char)
       - Export inferred types

    4. Create `src/app/api/auth/[...all]/route.ts`:
       ```typescript
       import { auth } from "@/lib/auth";
       import { toNextJsHandler } from "better-auth/next-js";
       export const { GET, POST } = toNextJsHandler(auth.handler);
       ```

    5. Create `src/app/(auth)/layout.tsx` -- minimal centered layout for auth pages. No nav chrome. Just a centered card container.

    6. Create `src/components/auth/sign-up-form.tsx` -- client component ("use client"):
       - Use React Hook Form with zodResolver and signUpSchema
       - Fields: name, email, password, companyName
       - On submit:
         a. Call `authClient.signUp.email({ email, password, name })`
         b. On success, call `authClient.organization.create({ name: companyName, slug: slugify(companyName) })`
         c. Then call `authClient.organization.setActive({ organizationId })` to set the new org as active
         d. Redirect to `/onboarding` using `useRouter().push`
       - Show loading state on button during submission
       - Display error messages from Better Auth (email already exists, etc.)
       - Link to sign-in page at bottom

    7. Create `src/app/(auth)/sign-up/page.tsx` -- Server Component wrapper that renders SignUpForm. Page title "Create your account".

    8. Create `src/components/auth/sign-in-form.tsx` -- client component ("use client"):
       - Use React Hook Form with zodResolver and signInSchema
       - Fields: email, password
       - On submit: call `authClient.signIn.email({ email, password })`
       - On success: redirect to `/dashboard` using `useRouter().push`
       - Show loading state, display errors
       - Link to sign-up page at bottom

    9. Create `src/app/(auth)/sign-in/page.tsx` -- Server Component wrapper that renders SignInForm. Page title "Sign in to HeavyLeads".

    IMPORTANT: All forms use shadcn/ui components (Button, Card, Input, Label). Use the shadcn Form component with React Hook Form integration where appropriate.
    IMPORTANT: nextCookies() MUST be the last plugin in the array (Pitfall 1 from research).
    IMPORTANT: All uses of headers(), cookies(), params, searchParams must be awaited (Next.js 16 requirement).
