    1. Create `src/lib/validators/onboarding.ts`:
       - `locationSchema`: hqAddress (string, min 5 chars, "Please enter a valid address")
       - `equipmentSchema`: equipmentTypes (array of strings, min 1, "Select at least one equipment type")
       - `radiusSchema`: serviceRadius (number, min 10 "Minimum 10 miles", max 500 "Maximum 500 miles")
       - `onboardingSchema`: z.object combining all three (for final submission validation)
       - Export `OnboardingFormData` type inferred from onboardingSchema
       - Import and re-export EQUIPMENT_TYPES from @/types

    2. Create `src/actions/onboarding.ts` -- Server Action ("use server"):
       - `completeOnboarding(data: OnboardingFormData)`:
         a. Get session via `auth.api.getSession({ headers: await headers() })` -- if no session or no activeOrganizationId, throw "Unauthorized"
         b. Validate data with onboardingSchema.parse()
         c. Call `geocodeAddress(data.hqAddress)` to get lat/lng
         d. Insert into companyProfiles table with organizationId from session, geocoded coords, equipment types, radius, onboardingCompleted: true
         e. revalidatePath("/") to clear cached redirects
         f. Return { success: true }
       - `geocodeAddress(address: string)` (not exported, internal helper):
         a. Read GOOGLE_MAPS_API_KEY from process.env
         b. If no key, log warning and return { lat: 0, lng: 0, formattedAddress: address } (graceful degradation for dev without API key)
         c. Fetch `https://maps.googleapis.com/maps/api/geocode/json?address=${encodeURIComponent(address)}&key=${apiKey}`
         d. Parse response, extract lat/lng from results[0].geometry.location
         e. Return { lat, lng, formattedAddress: results[0].formatted_address }
         f. On error, throw "Unable to geocode address. Please check the address and try again."
       IMPORTANT: Geocoding MUST be server-side only. API key never exposed to client (Pitfall 4).

       - Render: progress indicator (Step 1 of 3, Step 2 of 3, Step 3 of 3), current step component, navigation buttons (Back/Next/Complete)
       - "Next" button validates current step's fields before advancing:

       - "Complete" button on final step: calls handleSubmit -> completeOnboarding server action -> on success redirect to /dashboard via router.push

    2. Create `src/actions/settings.ts` -- Server Actions ("use server"):
       - `updateAccount(data: AccountSettingsInput)`:
         a. Get session, validate authenticated
         b. Validate data with accountSettingsSchema
         c. Call `auth.api.updateUser({ body: { name: data.name }, headers: await headers() })` -- Better Auth handles user updates
         d. If email changed, handle via Better Auth's email update flow (or note as future enhancement if Better Auth doesn't support inline email change)
         e. revalidatePath("/settings/account")
         f. Return { success: true }
       - `updateCompanyProfile(data: CompanySettingsInput)`:
         a. Get session, validate authenticated, get activeOrganizationId
         b. Check admin role: `const member = await auth.api.getActiveMember({ headers: await headers() })` -- if role is not "owner" and not "admin", throw "Only admins can update company profile"
         c. Validate data with companySettingsSchema
         d. If hqAddress changed from current value, re-geocode via geocodeAddress (import from onboarding actions or extract to shared util)
         e. Update companyProfiles where organizationId matches, set all fields + updatedAt: new Date()
         f. revalidatePath("/settings/company")
         g. Return { success: true }
       - `getCompanyProfile()`:
         a. Get session, get activeOrganizationId
         b. Query companyProfiles where organizationId matches
         c. Return profile data

    3. Create `src/app/(dashboard)/settings/layout.tsx`:
       - Server Component
       - Sidebar navigation within settings: "Account" link to /settings/account, "Company" link to /settings/company
       - Use tabs or vertical nav pattern with shadcn components

    4. Create `src/components/settings/account-form.tsx` -- client component ("use client"):
