<objective>
Create a Socrata SODA3 base adapter class and migrate Austin/Dallas permits to extend it, then expand the SAM.gov adapter with configurable per-industry NAICS codes and rate limiting.

Purpose: Generalize the permit scraping architecture so adding new Socrata cities is a simple configuration exercise (not a copy-paste of HTTP logic), and enable SAM.gov to serve industry-specific bid results via NAICS filtering. Both adapter types integrate p-queue rate limiting from Plan 01.

Output: SocrataPermitAdapter base class, refactored Austin/Dallas adapters, expanded SAM.gov adapter, comprehensive tests.
</objective>

           sourceName: string;
           jurisdiction: string;
           domain: string;           // e.g., 'data.austintexas.gov'
           datasetId: string;        // e.g., '3syk-w9eu'

             address: string;

       - Create abstract class `SocrataPermitAdapter implements ScraperAdapter`:
         - Constructor takes SocrataConfig
         - sourceId, sourceName, jurisdiction from config
         - sourceType = 'permit'
         - `scrape()` method:
           a. Import and get Socrata queue: `const queue = await getSocrataQueue()`
           b. Wrap the API call in `queue.add(async () => { ... })`
           c. Try SODA3 first: POST to `https://${domain}/api/v3/views/${datasetId}/query.json`
              - Body: `{ query: "SELECT * WHERE ${permitDateField} > '${dateStr}' ORDER BY ${permitDateField} DESC LIMIT 1000" }`
              - Headers: Content-Type application/json, X-App-Token if SOCRATA_APP_TOKEN env var set (always .trim())
           d. If SODA3 response not ok (403, 404, etc.): fall back to SODA2
           e. SODA2 fallback: GET `https://${domain}/resource/${datasetId}.json` with $where, $limit, $order params
           f. Parse response JSON, call abstract `mapRecords()` method
         - Abstract `mapRecords(data: Record<string, unknown>[]): RawLeadData[]` -- subclasses implement this
       - Import getSocrataQueue from '../api-rate-limiter'
