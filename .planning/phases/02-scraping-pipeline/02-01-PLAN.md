  <files>src/lib/db/schema/leads.ts, src/lib/db/schema/index.ts, src/lib/scraper/types.ts, src/lib/scraper/adapters/base-adapter.ts, src/lib/scraper/registry.ts, .gitignore, package.json</files>
  <action>
    1. Install dependencies: `npm install crawlee node-cron` and `npm install -D @types/node-cron`

    2. Create `src/lib/db/schema/leads.ts` — Define the leads table following the existing company-profiles pattern:
       - `id`: uuid PK with defaultRandom
       - `permitNumber`: text, notNull
       - `description`: text, nullable
       - `address`: text, notNull
       - `formattedAddress`: text, nullable (from geocoding)
       - `lat`: real, nullable
       - `lng`: real, nullable
       - `projectType`: text, nullable
       - `estimatedValue`: integer, nullable
       - `applicantName`: text, nullable
       - `permitDate`: timestamp, nullable
       - `sourceId`: text, notNull (adapter identifier e.g. "austin-tx-permits")
       - `sourceJurisdiction`: text, notNull (e.g. "Austin, TX")
       - `sourceUrl`: text, nullable
       - `scrapedAt`: timestamp, defaultNow, notNull
       - `createdAt`: timestamp, defaultNow, notNull
       - Indexes: `leads_source_permit_idx` unique composite on (sourceId, permitNumber) for dedup upserts, `leads_scraped_at_idx` on scrapedAt
       - NOTE on PostGIS: Drizzle ORM's `geometry()` column type requires the PostGIS extension to be enabled on the database. Since the Neon serverless driver may have compatibility issues with PostGIS geometry types (Open Question #4 from research), use plain `real` columns for lat/lng now. The PostGIS geometry column and GiST spatial index can be added in a migration once verified to work with the Neon driver. This is a safe fallback — lat/lng columns support Haversine-based queries for MVP volumes, and PostGIS can be layered on without schema changes. Store lat/lng as plain real columns, document the PostGIS upgrade path in a code comment.

    3. Update `src/lib/db/schema/index.ts` — Add `export * from "./leads";`

    4. Create `src/lib/scraper/types.ts` — Shared types:
       - `PipelineResult`: { sourceId: string, sourceName: string, recordsScraped: number, recordsStored: number, errors: string[] }
       - `PipelineRunResult`: { results: PipelineResult[], startedAt: Date, completedAt: Date }

    5. Create `src/lib/scraper/adapters/base-adapter.ts` — Define ScraperAdapter interface and Zod schema:
       - `rawPermitSchema` Zod object: permitNumber (string), description (string optional), address (string), projectType (string optional), estimatedValue (number optional), applicantName (string optional), permitDate (date optional/coercible), sourceUrl (string optional)
       - `RawPermitData` type inferred from schema
       - `ScraperAdapter` interface: readonly sourceId, readonly sourceName, readonly jurisdiction, scrape() returning Promise<RawPermitData[]>

    6. Create `src/lib/scraper/registry.ts` — Simple adapter registry:
       - `adapters` Map<string, ScraperAdapter>
       - `registerAdapter(adapter: ScraperAdapter)` — adds to map by sourceId
       - `getRegisteredAdapters(): ScraperAdapter[]` — returns all registered adapters
       - `clearAdapters()` — for testing

    7. Add `storage/` to `.gitignore` (Crawlee creates local storage directories)
