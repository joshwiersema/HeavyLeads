**Rewrite `src/lib/scraper/expiration.ts`:**

Replace the current `severity: "expired"` marking logic with actual DELETE operations that preserve bookmarked and interacted leads.

```typescript
import { db } from "@/lib/db";
import { leads } from "@/lib/db/schema/leads";
import { bookmarks } from "@/lib/db/schema/bookmarks";
import { leadStatuses } from "@/lib/db/schema/lead-statuses";
import { sql, lt, and, isNull, ne } from "drizzle-orm";

/** Number of days before a lead is eligible for deletion */
const EXPIRATION_DAYS = 45;

/** Batch size for deletion to avoid overwhelming the database */
const DELETE_BATCH_SIZE = 500;

export async function expireStaleLeads(): Promise<{ expired: number }> {
  const cutoffDate = new Date(Date.now() - EXPIRATION_DAYS * 24 * 60 * 60 * 1000);
  let totalExpired = 0;

  // Delete leads older than 45 days that:
  // 1. Have NO bookmark referencing them
  // 2. Have NO lead_status entry (meaning user never interacted)
  //
  // Use raw SQL for the NOT EXISTS subqueries since Drizzle ORM
  // doesn't have great support for correlated subqueries in DELETE.
  //
  // Process in batches to avoid long-running queries on Neon serverless.

  let deletedInBatch: number;
  do {
    const result = await db.execute(sql`
      DELETE FROM leads
      WHERE id IN (
        SELECT l.id FROM leads l
        WHERE l.scraped_at < ${cutoffDate}
        AND NOT EXISTS (
          SELECT 1 FROM bookmarks b WHERE b.lead_id = l.id
        )
        AND NOT EXISTS (
          SELECT 1 FROM lead_statuses ls WHERE ls.lead_id = l.id
        )
        LIMIT ${DELETE_BATCH_SIZE}
      )
    `);

    deletedInBatch = Number(result.rowCount ?? 0);
    totalExpired += deletedInBatch;

    console.log(`[expire] Batch deleted ${deletedInBatch} leads (total: ${totalExpired})`);
  } while (deletedInBatch === DELETE_BATCH_SIZE);

  return { expired: totalExpired };
}
```

Key design:
- **45-day cutoff** based on `scraped_at` (not created_at) -- this matches when the data was last confirmed fresh
- **NOT EXISTS bookmarks** -- any lead with a bookmark row is preserved regardless of age
- **NOT EXISTS lead_statuses** -- any lead a user has explicitly interacted with (viewed, contacted, won, lost) is preserved
- **Batch deletion** (500 per batch) to avoid Neon serverless query timeouts on large datasets
- **Loop** until fewer than batch size are deleted (all eligible leads processed)
- Related rows in `lead_sources`, `lead_enrichments` are cleaned up via `onDelete: "cascade"` FK constraints on those tables
