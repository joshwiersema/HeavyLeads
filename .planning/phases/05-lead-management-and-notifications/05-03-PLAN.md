Output: React Email template, digest generation logic, Resend integration, API route for triggering, and scheduler integration.

           ```typescript
           try {
             const { generateDigests } = await import("@/lib/email/digest-generator");
             const digestResult = await generateDigests();
             console.log(`[scheduler] Email digest: ${digestResult.sent} sent, ${digestResult.skipped} skipped, ${digestResult.errors} errors`);
           } catch (digestError) {
             console.error("[scheduler] Digest generation failed:", digestError instanceof Error ? digestError.message : digestError);
           }
           ```
       - The digest call is wrapped in its own try/catch so a digest failure does not affect the scraper pipeline status
       - Keep the API route as an alternative trigger point (for manual triggering via curl or external cron services like Vercel Cron)

    The API route serves as both: (1) a manual trigger endpoint for testing/debugging, and (2) a target for external cron services if deployed to a platform without node-cron support (e.g., Vercel serverless). The scheduler integration handles the primary daily trigger path.
