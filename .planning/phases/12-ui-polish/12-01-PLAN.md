<objective>
Add active-state navigation highlighting to both the desktop sidebar and mobile nav drawer so users always know where they are in the app.

Purpose: The desktop sidebar currently has zero active-state logic (all 4 links look identical regardless of current page). The mobile nav has active-state logic but uses an exact match for `/dashboard` that fails for nested routes like `/dashboard/leads/[id]`. This plan consolidates navigation config into a shared module and adds correct route matching to both navigation surfaces.

Output: Shared nav config (`nav-links.ts`), new client component (`sidebar-nav.tsx`), updated mobile nav, updated layout, and comprehensive tests for all route scenarios.
</objective>
