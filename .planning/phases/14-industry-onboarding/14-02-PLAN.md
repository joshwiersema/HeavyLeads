**Rewrite src/actions/onboarding.ts:**
Replace the existing `completeOnboarding` function. Keep it as a server action ("use server").

New signature: `completeOnboarding(data: WizardState): Promise<{ success: boolean; error?: string }>`.

Implementation:
1. Get session (same auth check as existing).
2. Validate that `data.industry` is one of the 5 valid industries.
3. Compose full address from `data.street, data.city, data.state, data.zip` using `composeAddress()`.
4. If `data.serviceAreaLat` and `data.serviceAreaLng` are set (from Places autocomplete), use them directly. Otherwise, call `geocodeAddress(fullAddress)` to get coordinates. If geocoding returns null coordinates, return error (same as existing pattern).
5. Update the organization's industry field: `db.update(organization).set({ industry: data.industry }).where(eq(organization.id, orgId))`.
6. Upsert into `organizationProfiles` with ALL fields from wizard state:
   - organizationId, hqAddress (formatted), hqLat, hqLng
   - serviceRadiusMiles: data.serviceRadiusMiles
   - equipmentTypes: data.specializations (reuse column for backward compat -- heavy_equipment specializations ARE equipment types)
   - specializations: data.specializations
   - serviceTypes: data.serviceTypes
   - certifications: data.certifications
   - targetProjectValueMin: data.minProjectValue
   - targetProjectValueMax: data.maxProjectValue
   - yearsInBusiness: data.yearsInBusiness
   - companySize: data.companySize
   - onboardingCompleted: true
   - updatedAt: new Date()
7. Use `.onConflictDoUpdate({ target: [organizationProfiles.organizationId], set: { ... } })` (same pattern as existing but with all new fields).
8. Call `revalidatePath("/")`.
9. Return `{ success: true }`.

Import from sql template: `import { sql } from "drizzle-orm"` -- use `sql\`excluded.column_name\`` for upsert values per existing project decision.

IMPORTANT: Do NOT use `sql\`excluded...\`` -- Drizzle's `.onConflictDoUpdate({ set: { field: value } })` is the correct pattern (already used in the existing action). The `excluded` pattern is only for raw SQL.
