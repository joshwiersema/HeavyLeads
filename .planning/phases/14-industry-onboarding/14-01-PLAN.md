Replace the existing `useState(0)` + `react-hook-form` wizard with a `useReducer`-based wizard.

Import `wizardReducer`, `initialWizardState` from `@/lib/onboarding/reducer`. Import `useWizardPersistence` from `@/lib/onboarding/use-wizard-persistence`. Import `WIZARD_STEPS` from `@/lib/onboarding/types`. Import `getStepSchema` from `@/lib/validators/onboarding`.

The wizard shell:
1. Calls `useReducer(wizardReducer, initialWizardState)` to get `[state, dispatch]`.
2. Calls `useWizardPersistence(state, dispatch)` to get `{ isHydrated }`.
3. Shows a loading spinner/skeleton until `isHydrated` is true.
4. Renders a Card with:
   - Header showing "Set up your company" title, step N of 6 label, step name from WIZARD_STEPS
   - Progress bar: 6 segments, filled up to currentStep (same pattern as existing but 6 segments)
   - Dynamic step component based on `state.currentStep` using a switch/map
   - Footer with Back/Next/Complete buttons
5. Back button: dispatches `{ type: "PREV_STEP" }`, disabled on step 0.
6. Next button: validates current step's fields using `getStepSchema(state.currentStep).safeParse(...)` on relevant state fields. If valid, dispatches `{ type: "NEXT_STEP" }`. If invalid, shows toast with first error.
7. On last step (Review), the "Complete" button calls the completion server action (to be wired in Plan 02 -- for now, show a toast "Onboarding complete!" and redirect to /billing).
8. Step components receive `state` and `dispatch` as props (not react-hook-form context).

Define a `WizardStepProps` interface: `{ state: WizardState; dispatch: React.Dispatch<WizardAction> }` and export it.

For steps 3-6 that will be built in Plan 02, render a placeholder `<div>` with the step name and "Coming in next plan" text. This allows the wizard to be navigated end-to-end for testing.
