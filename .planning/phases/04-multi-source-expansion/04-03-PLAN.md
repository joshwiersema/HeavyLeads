  <name>Task 1: Cross-source deduplication engine</name>
  <files>
    src/lib/scraper/dedup.ts
    tests/scraper/dedup.test.ts
  </files>
  <behavior>
    - Two leads within 0.1 miles with address similarity > 0.7 are merged (source references transferred to canonical lead, duplicate deleted)
    - Two leads within 0.1 miles with title similarity > 0.7 are merged
    - Two leads > 0.1 miles apart are NOT merged regardless of text similarity
    - Two leads within 0.1 miles but both address AND title similarity < 0.7 are NOT merged (distinct nearby projects)
    - Leads without coordinates (null lat/lng) are kept as-is (cannot dedup geographically)
    - When merging: lead_sources from the duplicate are transferred to the canonical lead, then the duplicate lead row is deleted
    - When merging: the canonical lead (existing, older) is kept; the newer duplicate is removed
    - normalizeText strips punctuation, lowercases, and trims whitespace
    - deduplicateNewLeads returns { merged: number, kept: number } counts
    - Threshold constants are exported for configurability
  </behavior>
  <action>
    Install string-similarity: `npm install string-similarity && npm install -D @types/string-similarity`

    1. Create src/lib/scraper/dedup.ts:
       - Import compareTwoStrings from "string-similarity"
       - Import haversineDistance from "@/lib/leads/queries"
       - Import db, leads table, leadSources table, and drizzle-orm operators (eq, and, sql, ne, isNotNull)
       - Export constants: PROXIMITY_THRESHOLD_MILES = 0.1, SIMILARITY_THRESHOLD = 0.7
       - Export function normalizeText(text: string | null): string — lowercase, strip non-alphanumeric except spaces, trim
       - Export function isLikelyDuplicate(a, b): boolean — takes two objects with {lat, lng, normalizedAddress, normalizedTitle}:
         * If either lacks coordinates, return false
         * Compute haversineDistance — if > PROXIMITY_THRESHOLD_MILES, return false
         * Compute addressSim = compareTwoStrings(a.normalizedAddress, b.normalizedAddress)
         * Compute titleSim = compareTwoStrings(a.normalizedTitle, b.normalizedTitle)
         * Return addressSim > SIMILARITY_THRESHOLD || titleSim > SIMILARITY_THRESHOLD
       - Export async function deduplicateNewLeads(newLeadIds: string[]): Promise<{merged: number, kept: number}>:
         * For each newLeadId, fetch the lead from DB
         * Skip if no lat/lng (increment kept, continue)
         * Find candidate leads: query leads table where id != newLeadId AND lat IS NOT NULL AND lng IS NOT NULL. To avoid scanning all leads, add a bounding box pre-filter: lat BETWEEN (lead.lat - 0.002) AND (lead.lat + 0.002) AND lng BETWEEN (lead.lng - 0.002) AND (lead.lng + 0.002). This approximates ~0.15 miles at mid-latitudes and avoids computing haversine for distant leads.
         * For each candidate, check isLikelyDuplicate
         * If match found, call mergeLeads(canonicalId=candidate.id, duplicateId=newLeadId)
         * mergeLeads: (1) Update leadSources rows where leadId=duplicateId to set leadId=canonicalId, (2) Delete the duplicate lead row. Use a transaction for atomicity.
         * Log each merge: `[dedup] Merged lead ${duplicateId} into ${canonicalId} (distance: X mi, similarity: X)`

    2. Create tests/scraper/dedup.test.ts:
       - Test normalizeText: "123 Main St." -> "123 main st", null -> ""
       - Test isLikelyDuplicate: same location + similar address -> true
       - Test isLikelyDuplicate: same location + similar title -> true
       - Test isLikelyDuplicate: same location + different address AND title -> false
       - Test isLikelyDuplicate: distant location + similar address -> false
       - Test isLikelyDuplicate: one null lat/lng -> false
       - For deduplicateNewLeads integration: mock db to test merge/keep logic
       - Test: merged count correct when duplicates found
       - Test: kept count correct when no duplicates
       - Test: leads without coordinates are kept (not merged)
  </action>
  <verify>

  </verify>
