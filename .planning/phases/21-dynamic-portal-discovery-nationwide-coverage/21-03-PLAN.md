```typescript
      id: string;        // dataset ID (e.g., "3syk-w9eu")

      columns_field_name: string[];
      columns_datatype: string[];
```

       ```typescript
         domain: string;

         name: string;
         datasetType: "permit" | "violation";
         city: string | null;
         state: string | null;
         jurisdiction: string | null;

         discoveredBy: "socrata-discovery";

       ```

    2. Define search queries array (run discovery for each):
       ```typescript
       const DISCOVERY_QUERIES = [
         { q: "building permits", datasetType: "permit" as const },
         { q: "construction permits", datasetType: "permit" as const },
         { q: "code violations", datasetType: "violation" as const },
         { q: "code enforcement", datasetType: "violation" as const },
       ];
       ```

    3. Export async function `discoverSocrataDatasets(): Promise<SocrataDiscoveryResult[]>`:
       - For each query in DISCOVERY_QUERIES:
         - Fetch `https://api.us.socrata.com/api/catalog/v1?q=${encodeURIComponent(query.q)}&only=datasets&limit=100&offset=0`
         - Page through results (increment offset by 100) up to 500 results per query (5 pages max)
         - Add 500ms delay between page fetches to be respectful
         - For each result, extract `resource.columns_field_name` (the actual column names in the dataset)
         - Call `inferFieldMapping(columns)` to get mapping and confidence
         - Skip datasets with confidence < 0.33 (fewer than 3 canonical fields mapped -- unlikely to be useful)
         - Extract city/state from domain name heuristic: parse "data.{city}.gov" or "data.{city}{state}.gov" patterns. Also check result.metadata.domain.
         - Build jurisdiction string: `"${city}, ${state}"` when both available
         - Determine applicableIndustries: permit datasets apply to all 5 industries ["heavy_equipment","hvac","roofing","solar","electrical"]. Violation datasets apply to ["hvac","roofing","electrical"] (trades that respond to code violations).
       - Deduplicate by domain+datasetId (same dataset may appear in multiple queries)
       - Return array of SocrataDiscoveryResult sorted by confidence descending
       - Log summary: `[socrata-discovery] Found ${count} datasets across ${domains} domains`

    4. Export helper function `parseCityStateFromDomain(domain: string): { city: string | null; state: string | null }`:
       - Pattern: "data.{cityname}.gov" -> extract city, try to infer state from city name
       - Pattern: "data.{city}{state}.gov" -> e.g., "data.austintexas.gov" (Austin, Texas)
       - Pattern: "{city}.data.socrata.com" -> extract city
       - Pattern: "datahub.{city}.gov" -> extract city
       - For well-known domains, hardcode mappings: "data.austintexas.gov" -> Austin, TX; "www.dallasopendata.com" -> Dallas, TX; etc.
       - Return null for city/state when pattern doesn't match
       - Apply toTitleCase to city name
