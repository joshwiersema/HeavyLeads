    1. Import ScraperAdapter, RawLeadData, SourceType from base-adapter. Import DataPortalConfig from generic-socrata-adapter (reuse the same interface). Import FieldMapping from field-mapper. Import buildPermitTitle, toTitleCase from utils.

    2. Define local interfaces for ArcGIS response types:
       ```typescript
       interface ArcGISFeature {
         type: string;
         properties: Record<string, unknown>;
         geometry?: {
           type: string;
           coordinates: number[];
         };
       }

       interface ArcGISFeatureCollection {
         type: string;
         features: ArcGISFeature[];
       }

       // For ArcGIS Feature Service query response (non-GeoJSON)
       interface ArcGISQueryResponse {
         features: Array<{
           attributes: Record<string, unknown>;
           geometry?: { x: number; y: number };
         }>;
       }
       ```

    3. Export class `GenericArcGISAdapter implements ScraperAdapter`:
       - Constructor takes `DataPortalConfig`
       - `sourceId` = `"portal-${config.domain}-${config.datasetId}"`
       - `sourceName` = `config.name`
       - `sourceType` = map config.datasetType: "permit" -> "permit", "violation" -> "violation", default -> "permit"
       - `jurisdiction` = `config.jurisdiction ?? (config.city && config.state ? \`${config.city}, ${config.state}\` : undefined)`

    4. `scrape()` method with two fetch strategies:
       a. **Primary: GeoJSON download** (matches Atlanta pattern):
          - URL: `https://${domain}/api/v3/datasets/${datasetId}/downloads/data?format=geojson&spatialRefId=4326`
          - Parse response as ArcGISFeatureCollection
          - Extract coordinates from feature.geometry.coordinates [lng, lat]
          - Extract fields from feature.properties using fieldMapping

       b. **Fallback: Feature Service query** (for portals that don't support GeoJSON download):
          - URL: `https://${domain}/arcgis/rest/services/.../FeatureServer/0/query?where=1%3D1&outFields=*&f=json&resultRecordCount=1000`
          - Actually construct as: `https://${domain}/api/v3/datasets/${datasetId}/query?where=1%3D1&outFields=*&f=json&resultRecordCount=1000`
          - Parse as ArcGISQueryResponse
          - Extract fields from feature.attributes
          - Extract coordinates from feature.geometry {x, y}

       c. Error handling: If primary GeoJSON fetch returns non-200, try fallback. If both fail, throw.

    5. Private `mapGeoJSONFeatures(features: ArcGISFeature[]): RawLeadData[]`:
       - For each feature, use fieldMapping to extract values from feature.properties
       - Extract lat/lng from feature.geometry.coordinates (GeoJSON = [lng, lat])
       - Parse estimatedValue as float, skip NaN
       - Parse date field via `new Date(...)`
       - Build title via `buildPermitTitle`
       - Apply `toTitleCase` to address
       - Set sourceUrl = `https://${domain}/api/v3/datasets/${datasetId}`
       - Return RawLeadData array

    6. Private `mapQueryFeatures(features: ArcGISQueryResponse["features"]): RawLeadData[]`:
       - Similar to above but extracts from feature.attributes instead of feature.properties
       - Coordinates from feature.geometry.x (lng) and feature.geometry.y (lat)
       - Same field mapping logic

    7. Limit results to 2000 features max (ArcGIS can return large datasets).
