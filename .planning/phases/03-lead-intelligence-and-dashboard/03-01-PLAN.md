    Create `src/lib/leads/queries.ts` with two main query functions using Drizzle ORM:

    1. `getFilteredLeads(params)` -- the main feed query:
       - Parameters: `{ hqLat: number, hqLng: number, serviceRadiusMiles: number, dealerEquipment: string[], radiusMiles?: number, equipmentFilter?: string[], limit?: number, offset?: number }`
       - `radiusMiles` defaults to `serviceRadiusMiles` if not provided
       - Build Haversine distance expression using the exact SQL from 03-RESEARCH.md Pattern 1 (with LEAST/GREATEST clamp to avoid acos domain errors)
       - WHERE clause: `leads.lat IS NOT NULL AND leads.lng IS NOT NULL AND haversine_distance <= radiusMiles`
       - SELECT: all lead columns plus `distance` as computed column
       - ORDER BY: `scrapedAt DESC` (recency first)
       - Apply `limit` (default 50) and `offset` (default 0)
       - After query, enrich each result in TypeScript:
         - Call `inferEquipmentNeeds(lead.projectType, lead.description)` for each lead
         - Call `scoreLead(...)` with the inferred equipment types, dealer equipment, distance, service radius, and estimated value
         - Call `getFreshnessBadge(lead.scrapedAt)`
         - Call `mapTimeline(lead.projectType, lead.description)`
       - If `equipmentFilter` is provided (non-empty array), post-filter: keep only leads where at least one inferred equipment type is in the filter array
       - Re-sort final results by score DESC (secondary sort after recency -- actually sort by score DESC as primary for relevance, use scrapedAt DESC as tiebreaker)
       - Return `EnrichedLead[]`

    2. `getLeadById(id: string)` -- single lead for detail view:
       - Query single lead by UUID id
       - Return null if not found
       - Enrich with inferEquipmentNeeds, mapTimeline (no scoring needed for detail, but include it anyway for consistency)
       - Does NOT require hqLat/hqLng params -- takes optional `{ hqLat?: number, hqLng?: number, serviceRadiusMiles?: number, dealerEquipment?: string[] }` for scoring context. If not provided, score defaults to 0 and distance to null.
       - Return `EnrichedLead | null`

    IMPORTANT: The Haversine WHERE clause must repeat the full expression (cannot reference SELECT alias in PostgreSQL). This is documented in 03-RESEARCH.md Pattern 1 critical note. The query optimizer will not compute it twice.

    IMPORTANT: Guard against NULL lat/lng with `isNotNull(leads.lat)` and `isNotNull(leads.lng)` in the WHERE clause (Pitfall 1 from research).

    Import the db client from `@/lib/db`, leads schema from `@/lib/db/schema/leads`, and use `sql`, `getTableColumns`, `and`, `isNotNull`, `eq`, `desc` from `drizzle-orm`.
