API approach: EPA Envirofacts REST API
- Primary endpoint: `https://data.epa.gov/efservice/ACRES_SITE_INFORMATION/JSON/rows/0:100`
  - The Envirofacts pattern is: `https://data.epa.gov/efservice/{TABLE_NAME}/JSON/rows/{start}:{end}`
  - If ACRES_SITE_INFORMATION fails (404), try alternate table names in order:
    1. `ACRES_SITE_INFORMATION`
    2. `ACRES_SITES`
    3. `sems.ACRES_SITE_INFORMATION`
  - Log which table worked for debugging
- If all table names fail, return [] with console.warn: "EPA Brownfields tables may have been renamed. Check Envirofacts API viewer."

Map response array to RawLeadData:
- externalId: `epa-bf-${item.SITE_ID || item.HANDLER_ID || index}`
- title: `Brownfield Site: ${toTitleCase(item.SITE_NAME || item.NAME || "Unknown")}`
- description: Build from available fields: site name, cleanup status, assessment type, acreage
  - Example: `EPA Brownfield site "${site_name}" - Status: ${status}. ${acres} acres in ${city}, ${state}.`
- city: item.CITY or item.SITE_CITY
- state: item.STATE or item.SITE_STATE
- lat: parseFloat(item.LATITUDE) if present and valid
- lng: parseFloat(item.LONGITUDE) if present and valid
- sourceUrl: `https://enviro.epa.gov/enviro/acres_frs_query.html?fac_search=site_epa_id&fac_value=${item.SITE_ID}`
- postedDate: new Date() (EPA data doesn't always have date fields -- use current date)
- sourceType: "brownfield"

Use `getEpaQueue()` rate limiter. Wrap in try/catch, return [] on failure.
