  <behavior>
    - getLatestPipelineRun(orgId) returns the most recent pipeline run for the org (or any global cron run)
    - getOrgPipelineStatus(orgId) returns { hasEverRun: boolean, isRunning: boolean, lastRun: PipelineRun | null }
    - When no pipeline has ever run for this org AND no global cron run exists, hasEverRun is false
    - When a run exists with status "running", isRunning is true
    - shouldAutoTrigger(orgId, leadCount) returns true when hasEverRun is false AND leadCount is 0
    - shouldAutoTrigger returns false when leads exist (even if pipeline never ran for this org specifically)
  </behavior>
  <action>

       - Import db, pipelineRuns, and Drizzle operators
       - `getLatestPipelineRun(orgId: string)`: query pipeline_runs where organizationId = orgId OR organizationId IS NULL (covers both org-specific and global cron runs), ordered by startedAt desc, limit 1
       - `getOrgPipelineStatus(orgId: string)`: calls getLatestPipelineRun, returns { hasEverRun: boolean, isRunning: boolean, lastRun }
       - `shouldAutoTrigger(orgId: string, leadCount: number)`: returns true only when getOrgPipelineStatus shows hasEverRun=false AND leadCount === 0. This is the first-login detection. Pure decision function wrapping the async status check.

       - Test: "shouldAutoTrigger returns false when a pipeline has run before" -- mock findFirst returning a completed run
       - Test: "getOrgPipelineStatus returns isRunning=true when latest run is running" -- mock findFirst returning a run with status="running"
