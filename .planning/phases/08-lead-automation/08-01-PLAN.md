    - checkRateLimit returns { allowed: false, nextAllowedAt: Date } when a run exists within the past hour for the org

    - Cron-triggered runs (organizationId = null) do NOT affect per-org rate limiting

  <action>
    1. Create `src/lib/db/schema/pipeline-runs.ts`:
       - pgTable "pipeline_runs" with columns: id (uuid PK defaultRandom), organizationId (text, nullable -- null for cron global runs), triggeredBy (text -- user ID or "cron"), triggerType (text notNull -- "cron" | "manual" | "first-login"), status (text notNull default "pending" -- "pending" | "running" | "completed" | "failed"), recordsScraped (integer nullable), recordsStored (integer nullable), errorMessage (text nullable), startedAt (timestamp defaultNow notNull), completedAt (timestamp nullable)
       - Add index on organizationId + startedAt for rate limit queries
       - Follow existing schema patterns (pgTable, uuid, text, timestamp from drizzle-orm/pg-core)

    2. Update `src/lib/db/schema/index.ts`: Add `export * from "./pipeline-runs";`

    3. Create `src/lib/scraper/rate-limit.ts`:
       - Export `checkRateLimit(orgId: string): Promise<{ allowed: boolean; nextAllowedAt?: Date }>`
       - Query pipeline_runs for the given orgId where startedAt >= 1 hour ago, ordered by startedAt desc, limit 1
       - If a recent run exists, return { allowed: false, nextAllowedAt: startedAt + 1 hour }
       - If no recent run, return { allowed: true }
       - Use existing Drizzle query patterns: `db.query.pipelineRuns.findFirst(...)` with `and()`, `eq()`, `gte()`, `desc()`

    4. Update `tests/setup.ts`: Add `process.env.CRON_SECRET = "test-cron-secret";`

       - Mock `@/lib/db` with vi.mock
       - Test: "allows run when no recent runs exist" -- mock findFirst returning null
       - Test: "blocks run when recent run exists within 1 hour" -- mock findFirst returning a run from 30 min ago
       - Test: "allows run when last run is older than 1 hour" -- mock findFirst returning null (gte filter handles this)
       - Test: "returns nextAllowedAt as 1 hour after last run's startedAt"

    Run `npx drizzle-kit push` after schema creation to sync the database.
