    Create all 9 migration SQL files manually. Do NOT use `drizzle-kit generate` for this phase because the table rename (company_profiles -> organization_profiles) would be interpreted as drop + create, destroying data. All migrations are hand-written custom SQL.

    Also create the `src/lib/db/migrations/meta/_journal.json` file that Drizzle Kit uses to track migration state. This is needed for `drizzle-kit migrate` to work. The journal format is:
    ```json
    {
      "version": "7",
      "dialect": "postgresql",
      "entries": [
        { "idx": 0, "version": "7", "when": 1710600000000, "tag": "0000_create_postgis_extension", "breakpoints": true },
        { "idx": 1, "version": "7", "when": 1710600001000, "tag": "0001_rename_company_profiles", "breakpoints": true },
        ...
      ]
    }
    ```
    Use incrementing timestamps for the `when` field.

    **Migration 0: PostGIS Extension**
    ```sql
    CREATE EXTENSION IF NOT EXISTS postgis;
    ```

    **Migration 1: Rename company_profiles to organization_profiles**
    ```sql
    ALTER TABLE "company_profiles" RENAME TO "organization_profiles";
    ```
    This is the highest-risk migration. It MUST be ALTER TABLE RENAME, never DROP + CREATE.

    **Migration 2: Add columns to organization_profiles**
    ```sql
    ALTER TABLE "organization_profiles" ADD COLUMN "specializations" text[];
    ALTER TABLE "organization_profiles" ADD COLUMN "service_types" text[];
    ALTER TABLE "organization_profiles" ADD COLUMN "certifications" text[];
    ALTER TABLE "organization_profiles" ADD COLUMN "target_project_value_min" integer;
    ALTER TABLE "organization_profiles" ADD COLUMN "target_project_value_max" integer;
    ALTER TABLE "organization_profiles" ADD COLUMN "years_in_business" integer;
    ALTER TABLE "organization_profiles" ADD COLUMN "company_size" text;
    ```
    All nullable -- no NOT NULL constraints, no defaults needed.

    **Migration 3: Add industry to organization table + backfill**
    ```sql
    ALTER TABLE "organization" ADD COLUMN "industry" text DEFAULT 'heavy_equipment';
    UPDATE "organization" SET "industry" = 'heavy_equipment' WHERE "industry" IS NULL;
    ```
    The DEFAULT ensures new orgs without explicit industry get heavy_equipment. The UPDATE backfills existing orgs.

    **Migration 4: Add new columns to leads**
    ```sql
    ALTER TABLE "leads" ADD COLUMN "content_hash" text;
    ALTER TABLE "leads" ADD COLUMN "applicable_industries" text[] DEFAULT '{}';
    ALTER TABLE "leads" ADD COLUMN "value_tier" text;
    ALTER TABLE "leads" ADD COLUMN "severity" text;
    ALTER TABLE "leads" ADD COLUMN "deadline" timestamp;
    CREATE UNIQUE INDEX "leads_content_hash_idx" ON "leads" ("content_hash") WHERE content_hash IS NOT NULL;
    ```

    **Migration 5: Add PostGIS geometry column + GiST index + backfill**
    ```sql
    ALTER TABLE "leads" ADD COLUMN "location" geometry(Point, 4326);
    CREATE INDEX "leads_location_gist_idx" ON "leads" USING gist ("location");
    UPDATE "leads" SET "location" = ST_SetSRID(ST_MakePoint("lng", "lat"), 4326)
      WHERE "lat" IS NOT NULL AND "lng" IS NOT NULL AND "location" IS NULL;
    ```
    MUST come after Migration 0 (PostGIS extension). Note: MakePoint takes (x=lng, y=lat).

    **Migration 6: Add bookmark columns**
    ```sql
    ALTER TABLE "bookmarks" ADD COLUMN "notes" text;
    ALTER TABLE "bookmarks" ADD COLUMN "pipeline_status" text DEFAULT 'saved';
    ```

    **Migration 7: Create lead_enrichments table**
    ```sql
    CREATE TABLE "lead_enrichments" (
      "id" uuid PRIMARY KEY DEFAULT gen_random_uuid(),
      "lead_id" uuid NOT NULL REFERENCES "leads"("id") ON DELETE CASCADE,
      "enrichment_type" text NOT NULL,
      "data" text NOT NULL,
      "source" text,
      "fetched_at" timestamp DEFAULT now() NOT NULL,
      "expires_at" timestamp
    );
    CREATE INDEX "lead_enrichments_lead_id_idx" ON "lead_enrichments" ("lead_id");
    ```

    **Migration 8: Create scraper_runs table**
    ```sql
    CREATE TABLE "scraper_runs" (
      "id" uuid PRIMARY KEY DEFAULT gen_random_uuid(),

      "completed_at" timestamp
    );
    CREATE INDEX "scraper_runs_pipeline_run_idx" ON "scraper_runs" ("pipeline_run_id");
    ```
