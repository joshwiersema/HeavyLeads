3. Create `getFilteredLeadsWithCount(params)` that accepts GetFilteredLeadsParams plus `page: number` and `pageSize: number`. This function:
   - Runs the same Haversine-filtered SQL query as getFilteredLeads but WITHOUT FETCH_MULTIPLIER (removes the `limit * 4` over-fetch) and WITHOUT SQL-level limit/offset
   - Applies the same enrichment, equipment filtering, and score sorting as getFilteredLeads

   - Returns `{ leads: EnrichedLead[], totalCount: number, page: number, totalPages: number }`
   - Critical: the Haversine WHERE clause already limits results to within-radius leads (typically <5000 for any reasonable radius), so removing FETCH_MULTIPLIER and fetching all within radius is acceptable per research/out-of-scope notes

   - Guards: `if (ids.length === 0) return [];` (prevents invalid SQL from empty inArray)
   - Queries `db.select().from(leads).where(inArray(leads.id, ids))`
   - Maps each row through `enrichLead(row, params)`
   - Returns `EnrichedLead[]`
   - Import `inArray` from `drizzle-orm`
