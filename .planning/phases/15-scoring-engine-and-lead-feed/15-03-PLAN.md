**Create `score-breakdown.tsx`:**

A client component (needs no data fetching -- receives props) that displays the 5-dimension score breakdown visually.

Props: `{ scoring: ScoringResult }`

Layout:
- Large score number at top (same green/yellow/red color coding as lead card)
- Below: 5 horizontal bars, one per dimension. Each bar shows:
  - Dimension name (Distance, Relevance, Value, Freshness, Urgency) on the left
  - A thin progress bar (width proportional to score/maxScore) with color (green if >70% of max, yellow if >40%, gray otherwise)
  - Score/max text on the right (e.g., "20/25")
  - Below the bar: the dimension's reasons as muted text (join with "; ")
- Below all bars: "Match Reasons" heading with a bulleted list of `scoring.matchReasons` if any

Use Tailwind only -- no chart library needed. The progress bar is a simple `<div>` with `style={{ width: `${(score/maxScore)*100}%` }}`.

**Create `enrichment-cards.tsx`:**

A server component that receives parsed enrichment data as props (the parent page queries and parses the JSON). Since enrichments may not exist yet (no scrapers populating them yet), this component gracefully shows nothing when data is empty.

Props:
```typescript
interface EnrichmentCardsProps {
  weather: { summary: string; alerts: string[]; temperature?: number; conditions?: string } | null;
  property: { type?: string; yearBuilt?: number; sqft?: number; owner?: string; zoning?: string } | null;
  incentives: { name: string; amount: string; deadline?: string; url?: string }[] | null;
}
```

For each non-null enrichment, render a Card:
- **Weather card**: CloudSun icon, summary text, alert badges if any
- **Property card**: Home icon, details as a definition list (type, year built, sqft, owner, zoning)
- **Incentives card**: DollarSign icon, list of incentive programs with name, amount, deadline, and link

When all are null, render nothing (the section is hidden). This is the expected initial state since enrichment scrapers are not yet built (Phase 16+).

**Create `similar-leads.tsx`:**

A server component that receives `leadId`, `lat`, `lng`, and `orgId` as props. It queries leads in the same geographic area (within 25 miles using haversineDistance) that have different `applicableIndustries` from the current lead, limited to 5 results. For each, it shows a mini lead card (title, distance, projectType badge) linked to its detail page.

Implementation:
1. Import `db`, `leads` schema, `sql`, `ne` (not-equal) from drizzle
2. Query leads WHERE lat/lng within 25 miles of the given lat/lng (use Haversine SQL expression from queries.ts pattern), AND `leads.id != leadId`, AND `leads.applicableIndustries` does not overlap with the current lead's industries (or: just fetch nearby leads sorted by recency, excluding the current one -- simpler and still useful since the user can discover cross-industry leads)
3. Limit to 5
4. Render as a simple list of linked cards

If no similar leads found, render "No similar leads in this area" muted text.
