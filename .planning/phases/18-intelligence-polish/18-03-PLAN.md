       export const notificationPreferences = pgTable("notification_preferences", {
         id: uuid("id").primaryKey().defaultRandom(),
         userId: text("user_id").notNull().unique(),
         dailyDigest: boolean("daily_digest").default(true).notNull(),
         weeklySummary: boolean("weekly_summary").default(true).notNull(),
         // Future: stormAlerts, marketingEmails, etc.

    3. Create `src/lib/email/unsubscribe.ts`:
       - `generateUnsubscribeToken(userId: string, emailType: "daily_digest" | "weekly_summary"): string`
         - Creates HMAC-SHA256 of `userId:emailType` using `(process.env.UNSUBSCRIBE_SECRET ?? process.env.CRON_SECRET ?? "fallback-secret").trim()` as key
         - Returns base64url-encoded JSON: `{ userId, emailType, sig }` (where sig is the HMAC)
         - Token does NOT expire (CAN-SPAM requires permanent unsubscribe)
       - `validateUnsubscribeToken(token: string): { userId: string; emailType: string } | null`
         - Decodes base64url, verifies HMAC signature matches
         - Returns parsed payload or null
       - `unsubscribeUser(userId: string, emailType: string): Promise<void>`
         - Upserts notification_preferences: if row exists, update the emailType column to false; if not, insert with that column false and others defaulted
         - Uses ON CONFLICT (userId) DO UPDATE
       - `isSubscribed(userId: string, emailType: "daily_digest" | "weekly_summary"): Promise<boolean>`
         - Checks notification_preferences for the user
         - Returns true if no row exists (default subscribed) or if the column is true

    4. Create `src/components/emails/email-layout.tsx`:
       - Shared React Email layout component used by all email templates
       - Props: `{ children: React.ReactNode; industry?: string; unsubscribeUrl?: string; previewText?: string }`
       - Industry-specific header colors:
         - heavy_equipment: "#1e40af" (blue-800, current)
         - hvac: "#0d9488" (teal-600)
         - roofing: "#dc2626" (red-600)
         - solar: "#d97706" (amber-600)
         - electrical: "#4f46e5" (indigo-600)
         - default/unknown: "#1e40af"
       - Brand text: "LeadForge" (not HeavyLeads -- rebranding)
       - CAN-SPAM footer:
         - Unsubscribe link (when unsubscribeUrl provided): "Unsubscribe from these emails"
         - Physical address line: "LeadForge | [City, State]" (placeholder, user can configure later)
         - "You are receiving this email because you signed up for LeadForge."
       - Uses same inline style patterns as existing email templates

    5. Create `src/app/api/unsubscribe/route.ts`:
       - GET handler (one-click unsubscribe per CAN-SPAM / RFC 8058)

       - If invalid: return 400 JSON error
       - Also support POST for List-Unsubscribe-Post header (RFC 8058 requirement)
