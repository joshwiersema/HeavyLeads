    3. Add the `buildCheckoutSessionParams` function export that encapsulates the conditional setup fee logic:
       - If `plan.freeTrial` exists AND `subscription.trialStart` is null/undefined -> return `{}` (trial checkout, no setup fee)
       - If `subscription.trialStart` exists AND no `subscription.stripeSubscriptionId` -> return `{ params: { line_items: [{ price: plan.priceId, quantity: 1 }, { price: PRICES.setupFee, quantity: 1 }] } }` (post-trial first-time paid)

    IMPORTANT per research:
    - Do NOT include setup fee in trial checkout -- Stripe charges one-time items immediately even during trial
    - Return `{ params: { line_items: [...] } }` shape (not bare `{ line_items }`) per plugin API
    - When returning custom line_items, ALWAYS include the recurring subscription price alongside setup fee (line_items override replaces defaults)
    - getTrialStatus uses Math.ceil for daysRemaining so "5 hours left" shows as "1 day"
